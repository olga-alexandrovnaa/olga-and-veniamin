# Сайт приглашения на свадьбу

Одностраничный сайт с приглашением, обратным отсчётом и опросом гостей. Данные хранятся в Google Таблице.

## Технологии

- React 19, TypeScript, Vite
- CSS (без UI-библиотек), адаптивная вёрстка
- Чтение гостей — через открытый API (таблица «доступна по ссылке»), без настройки переменных
- Запись в таблицу — через Google Apps Script (по желанию)

## Запуск и тест на localhost

Переменные окружения **не нужны**.

```bash
npm install
npm run dev
```

Откройте в браузере: **http://localhost:3000/КОД**, где **КОД** — любое значение из колонки B вашей таблицы (например `http://localhost:3000/123123`). Сайт подставит ФИО из таблицы и покажет приглашение.

Сборка: `npm run build`. Превью: `npm run preview`.

## GitHub Pages

Проект настроен для публикации на GitHub Pages. После пуша в ветку `main`:

1. В репозитории откройте **Settings** → **Pages**.
2. В разделе **Build and deployment** выберите **Source: GitHub Actions**.
3. После первого пуша дождитесь завершения workflow **Deploy to GitHub Pages** (вкладка **Actions**).

Сайт будет доступен по адресу: **https://olga-alexandrovnaa.github.io/olga-and-veniamin/**  
Ссылки для гостей: `https://olga-alexandrovnaa.github.io/olga-and-veniamin/КОД` (например `/olga-and-veniamin/123123`).  
Печать приглашений: `https://olga-alexandrovnaa.github.io/olga-and-veniamin/papers`.

## Настройка таблицы Google

### Шаг 1. Заголовки и колонки

В **первой строке** таблицы задайте заголовки **точно в таком порядке** (каждая колонка — свой столбец):

| A | B | C | D | E | F | G | H | I | J |
|---|---|---|---|---|---|---|---|---|---|
| **ФИО** | **КОД** | **Подтвердили** | **Сообщение молодым** | **С парой** | **Предпочтения по алкоголю** | **Во сколько придут** | **Трансфер** | **Имя пары** | **Ограничения по еде** |

- **A (ФИО)** — имя гостя или пары.
- **B (КОД)** — уникальный код для ссылки (например `123123`). Ссылка для гостя: `https://ваш-сайт.ru/123123`.
- **C (Подтвердили)** — заполняется сайтом при нажатии «Подтвердить участие».
- **D (Сообщение молодым)** — текст из поля рядом с кнопкой подтверждения.
- **E–J** — ответы из опроса (заполняются при отправке формы).

Со второй строки — данные гостей: ФИО и КОД заполняете вы, остальные колонки — сайт.

### Шаг 2. Доступ для чтения

Чтобы сайт подставлял ФИО по коду: **Настройки доступа** → «Просматривать могут все, у кого есть ссылка».

## Сохранение ответов в таблицу (по желанию)

Кнопки «Подтвердить участие» и «Отправить» опрос будут сохранять данные в таблицу только после развёртывания Apps Script:

1. В таблице: **Расширения** → **Apps Script**, вставьте код из `apps-script/Code.gs`, сохраните, разрешите доступ.
2. **Развернуть** → **Новое развёртывание** → тип **Веб-приложение** («выполнять от имени: я», «все пользователи»). Скопируйте URL.
3. В проекте откройте `src/config/survey.ts` и вставьте URL в `APP_SCRIPT_URL: '...'` (сейчас там пустая строка).

Если `APP_SCRIPT_URL` не заполнен, сайт всё равно работает: гости видят приглашение и опрос, но подтверждения и ответы не записываются в таблицу.

**Важно:** порядок колонок в таблице должен совпадать с таблицей выше. Если нужно добавить свои вопросы — добавьте колонку в таблицу, затем в `src/config/survey.ts` — новый элемент в `SURVEY_QUESTIONS` с нужным `column` (номер столбца), и в `apps-script/Code.gs` — новую пару в объекте `SURVEY_COLUMNS`.

## Редактирование контента

- **Тексты приглашения** — в `src/pages/Invitation.tsx`, объект `INVITATION`.
- **Дата обратного отсчёта** — в `src/components/Countdown.tsx`, константа `TARGET_DATE`.
- **Вопросы опроса и колонки** — в `src/config/survey.ts`: массив `SURVEY_QUESTIONS` и при необходимости `SHEET_CONFIG`, `SHEET_NAME`.
